## 1.1 Presentation of Capstone

In our third year, we are required to complete a capstone project. The capstone is an academic requirement designed to help students apply theoretical knowledge and technical skills to a real-world problem by delivering a complete system with clear requirements, design, implementation, and documentation.

For Modula POS, the capstone objectives are implemented through:
- problem analysis and requirement definition,
- system design (architecture, data model, and module boundaries),
- implementation (frontend and backend),
- testing/evaluation,
- documentation and reporting.

---

## 1.1.1 Objective of Capstone

The main objectives of this capstone are:
- to integrate knowledge from software engineering, system analysis, databases, and project management into a coherent system,
- to design and implement a functional solution that addresses a real operational problem,
- to practice professional development workflows, including teamwork, communication, time management, and iterative refinement,
- to document the system in a way that supports implementation, verification, and future extension.
